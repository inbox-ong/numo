# Numo (Root Node Repository)

![Logo Numo](site/images/logo.svg)

**Bem-vindo ao respositório central do Numo.** 
O Numo é uma Infraestrutura Pública Digital (DPI) focada em Liquidação Bruta em Tempo Real (RTGS). Foi desenhado de forma puramente open source para que governos, provedores de pagamentos e Bancos Centrais possam possuir um núcleo soberano ("A Caixa Clara") sem serem reféns de *Vendor Lock-in* corporativo.

Este repositório principal atua como a árvore de conhecimento central: agrupa a governança de engenheiros e consolida metodologias antes de ramificarmos para sub-repósitos autônomos de rede.

## 🤝 Comunidade & Como Contribuir
Desenvolver código estatal exige rigorosidade matemática e debate civil aberto. Toda a cultura da Comunidade adere fielmente aos princípios do *The Open Source Way* Framework.

### 1. Modelos Fundacionais
- 📚 [**Guia de Contribuição (Contributing)**](docs/CONTRIBUTING.md): Se você codifica em Go/Rust ou formula regras de negócio, entenda como transitará do Status de Especialista até o envio do seu primeiro Pr. Leia imediatamente.
- ⚖️ [**A Governança do Projeto (Open Community)**](docs/GOVERNANCE.md): Como trabalhamos para promover de forma justa a liderança comitê baseada em mérito e código.
- 🛑 [**Código de Conduta Expresso**](docs/CODE_OF_CONDUCT.md): Política rígida "Zero Tolerância" a comportamentos antiéticos ou trollbacks; incluindo Mecanismos Rápidos de Denúncia.

### 2. Onde Nos Encontrar (Canais Oficiais)
Como exigido por documentações corporativas sérias, o debate de arquitetura e direção de produto (Features futuras) nunca é decidido offline em privado. Ele reside publicamente em:
- **Requisições Arquiteturais Baseadas em Consenso (RFCs):** A seção `Issues` baseada na Tag [RFC] aqui no GitHub.
- **Micro-Decisões Assíncronas:** A lista de discussão oficial via e-mail (se cadastre em numo-dev-list@...).
- **Suporte Geral (Dúvidas/Onboarding):** GitHub Discussions & Canal `#numo-onboarding` no Chat Comunitário Institucional.

> **💡 Dica "Good First Issue":** Se quer começar logo a mexer no código ou no doc, filtre em `Issues` pela tag `good-first-issue`!
